# MEMORY.md

Human-readable project memory. This complements, but does not replace, git history and per-unit `STATUS.md`/`LEARNING.md` files.

## 2026-07-06 — Startup spec finalized

The startup spec was reviewed and rewritten to make the project goal unambiguous: practical learning of LLM technology.

Key decisions:

- The project should not become an agent-authored "self-successful" nano LLM.
- Future work should be organized around durable Learning Units.
- `CONTEXT.md` is the canonical glossary, not a general project notebook.
- `AGENTS.md` holds agent operating rules.
- `ROADMAP.md` holds course sequence and active unit state.
- `LEARNING.md` records the user's predictions, attempts, review notes, and reflections.
- Agent guardrails should be lightweight warnings and reviews, not enforcement.
